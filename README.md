# Research Software Engineer Consulting - Strategy & Framework

## Overview
This repository contains the complete framework for providing effective Research Software Engineering (RSE) consultations to PhD students and postdocs.

## Purpose
To deliver high-quality, consistent, and impactful consulting services focused on:
- Code review and optimization
- Software architecture and design
- Best practices and training

## Session Format
- **Duration**: 45 minutes
- **Mode**: Remote, Individual
- **Types**: One-time sessions or ongoing support

## Quick Start
1. When contacted by a new client, send them the [Client_Intake_Form.md](Client_Intake_Form.md)
2. Review their submission and complete [Pre_Consultation_Checklist.md](Pre_Consultation_Checklist.md)
3. During session, follow [During_Consultation_Workflow.md](During_Consultation_Workflow.md)
4. After session, complete [Post_Consultation_Checklist.md](Post_Consultation_Checklist.md)
5. For code reviews, use [Python_Code_Review_Checklist.md](Python_Code_Review_Checklist.md)
