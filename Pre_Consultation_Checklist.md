# Pre-Consultation Checklist

**Client Name**: _________________________________  
**Date**: _________________________________  
**Session Type**: _________________________________

---

## 1. Administrative Setup (Day Before)

- [ ] **Confirm appointment** via email
  - Include meeting link/details
  - Remind them to share code if they haven't
  - Confirm time zone if remote
  
- [ ] **Review intake form** thoroughly
  - Understand their research context
  - Note their experience level
  - Identify session goals
  - Flag any special requirements

- [ ] **Verify meeting logistics**
  - Test video conferencing link
  - Check screen sharing capability
  - Prepare recording setup (if permission granted)
  - Have backup communication method ready

---

## 2. Code Review (If Code Provided)

- [ ] **Download and organize client code**
  - Clone repository or extract ZIP file
  - Create dedicated folder for this consultation
  - Note project structure at first glance

- [ ] **Initial code exploration (15-20 min)**
  - [ ] Read README or documentation
  - [ ] Identify entry points and main scripts
  - [ ] Note overall structure and organization
  - [ ] Check for obvious issues or concerns
  - [ ] List questions to ask during session

- [ ] **Test code locally (if feasible)**
  - [ ] Set up virtual environment
  - [ ] Install dependencies (check requirements.txt, setup.py, etc.)
  - [ ] Try to run main script(s)
  - [ ] Note any errors or warnings
  - [ ] Document any setup challenges

- [ ] **Run basic analysis tools (if time permits)**
  - [ ] Check for syntax errors: `python -m py_compile script.py`
  - [ ] Run linter: `pylint` or `flake8`
  - [ ] Check code formatting: `black --check`
  - [ ] Note any patterns or issues

- [ ] **Prepare initial notes**
  - [ ] List positive aspects to highlight
  - [ ] Identify 3-5 priority improvement areas
  - [ ] Note quick wins (easy fixes with high impact)
  - [ ] Prepare example refactorings if relevant

---

## 3. Technical Preparation

- [ ] **Prepare your environment**
  - [ ] Python environment ready with common packages
  - [ ] Code editor/IDE open and configured
  - [ ] Terminal ready for demonstrations
  - [ ] Git configured if version control discussion expected

- [ ] **Gather relevant resources**
  - [ ] Code examples for common patterns
  - [ ] Template repositories if relevant
  - [ ] Documentation links bookmarked
  - [ ] Best practices guides ready to share

- [ ] **Create session-specific materials folder**
  - [ ] Folder for code examples to create
  - [ ] Notes document ready
  - [ ] Screen capture ready if needed

---

## 4. Content Preparation (Based on Session Type)

### For Code Review Sessions
- [ ] Annotate problem areas in code (privately)
- [ ] Prepare 2-3 concrete examples of improvements
- [ ] Identify most critical issues to address first
- [ ] Prepare positive feedback points
- [ ] Have refactoring examples ready

### For Architecture Consultation
- [ ] Review common design patterns for their use case
- [ ] Prepare template project structure
- [ ] Have example module organization ready
- [ ] Diagram tools ready if needed
- [ ] Prepare scalability discussion points

### For Training/Best Practices
- [ ] Prepare hands-on examples
- [ ] Create simple exercises if time permits
- [ ] Have live coding example ready
- [ ] Prepare step-by-step guides
- [ ] Test any tools or commands you'll demonstrate

### For Debugging/Troubleshooting
- [ ] Review error messages shared in advance
- [ ] Research similar issues if needed
- [ ] Prepare debugging workflow demonstration
- [ ] Have logging/debugging tools ready
- [ ] Prepare systematic troubleshooting approach

### For Follow-up Sessions
- [ ] Review previous session notes
- [ ] Check if action items from last time are complete
- [ ] Prepare to address previous questions
- [ ] Note progress since last session
- [ ] Prepare next-level content

---

## 5. Mental Preparation

- [ ] **Review session objectives**
  - What does the client want to achieve?
  - What are realistic goals for 45 minutes?
  - What would make this session successful?

- [ ] **Prepare opening questions**
  - Questions to clarify goals
  - Questions to understand context better
  - Questions to gauge their understanding

- [ ] **Plan time allocation**
  - 5-10 min: Introduction and goal setting
  - 25-35 min: Active work
  - 5-10 min: Summary and next steps

- [ ] **Set personal goals**
  - Key concepts to communicate
  - Skills to transfer
  - Confidence to build

---

## 6. Materials Checklist

**Have ready to share:**
- [ ] Relevant code examples
- [ ] Best practices documents
- [ ] Resource links
- [ ] Template files or projects
- [ ] Tool installation guides

**Have ready for yourself:**
- [ ] Session notes template
- [ ] Code review checklist (if applicable)
- [ ] Client intake form for reference
- [ ] Previous session notes (if follow-up)
- [ ] Timer to track time

---

## 7. Final Check (15 min before session)

- [ ] **Technical check**
  - [ ] Close unnecessary applications
  - [ ] Silence notifications
  - [ ] Test microphone and camera
  - [ ] Test screen share
  - [ ] Have client code open and ready
  - [ ] Have backup plans for technical issues

- [ ] **Mental check**
  - [ ] Clear any distractions
  - [ ] Review key goals one more time
  - [ ] Prepare welcoming opening
  - [ ] Get water/coffee
  - [ ] Take a deep breath!

- [ ] **Materials check**
  - [ ] All tabs/files ready
  - [ ] Note-taking ready
  - [ ] Links and resources in one place
  - [ ] Examples prepared and tested

---

## Quick Reference

**If client didn't share code:**
- Prepare to do live code review during session
- Have screen-sharing workflow ready
- Prepare to take notes on code structure quickly
- May need to schedule follow-up for deeper review

**If client is new to programming:**
- Prepare simpler examples
- Avoid jargon or explain terms carefully
- Focus on fundamental concepts
- Be extra encouraging and patient

**If urgent/deadline-driven:**
- Prioritize quick wins ruthlessly
- Focus on what's most critical
- Prepare to provide follow-up resources
- Consider offering brief follow-up check-in

**If exploring new tools/concepts:**
- Have official documentation ready
- Prepare installation walkthrough
- Test tools yourself beforehand
- Have troubleshooting resources ready
